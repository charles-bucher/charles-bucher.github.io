# Contributing to charles-bucher.github.io

We welcome contributions that improve documentation clarity, reproducibility, and the overall presentation of this project.

## How to Contribute

- **Fork the repository** and create your branch from `main`.
- **Write clear commits** that explain the change (e.g., `docs: update CONTRIBUTING guidelines`).
- **Add documentation** for any new features, fixes, or workflows.
- **Test locally** before submitting a pull request.
- **Open a Pull Request (PR)** with a detailed description of:
  - What problem you solved
  - How you solved it
  - Any screenshots or references that help reviewers

## Code Style

- Use **2 spaces** for indentation.
- Keep Markdown clean and consistent.
- Use descriptive headings and lists for readability.

## Reporting Issues

- Use the **Issues tab** to report bugs or request features.
- Include steps to reproduce and any relevant screenshots or logs.

## Community Guidelines

- Be respectful and constructive.
- Focus on clarity and reproducibility.
- Share insights that empower others.

Thank you for helping make this project a professional and accessible resource!
